---
title: "POST /v1/validations — Start Phone Number Validation"
sidebarTitle: "Start Validation"
description: "POST /v1/validations initiates phone number OTP verification, sending a 6-digit code via SMS and returning a validation token."
---

Call this endpoint to begin verifying a phone number. The API sends a 6-digit code to the supplied phone number via SMS and returns a `token` that you pass to subsequent verification calls. Each validation session is valid for approximately 10 minutes.

## Endpoint

```
POST https://verify-api.smsmanager.com/v1/validations
```

## Authentication

Include your API key in the `X-API-Key` header on every request.

```http
X-API-Key: <your-api-key>
```

## Request body

<ParamField body="phoneNumber" type="string" required>
  The phone number to validate. Provide digits only with no spaces, dashes, or leading `+` — include the country code (e.g., `14155550100` for a US number). Must match `^[1-9]\d{6,14}$`.
</ParamField>

<ParamField body="turnstileToken" type="string">
  A Cloudflare Turnstile token. Required only if your tenant has the `requireTurnstile` option enabled. Obtain this token from the Turnstile widget rendered on your front end.
</ParamField>

<ParamField body="payload" type="object">
  An optional key-value object you want to attach to this validation. The payload is returned in callbacks, making it easy to correlate a validation with a user session or order.

  ```json
  { "userId": "user123", "source": "checkout" }
  ```
</ParamField>

## Response fields

<ResponseField name="token" type="string">
  A unique validation token. Pass this as the `{token}` path parameter when you call `/v1/validations/{token}/verify`.
</ResponseField>

<ResponseField name="status" type="string">
  Current status of the validation. At this stage the value will be `pending` (SMS not yet delivered) or `code_sent` (SMS delivered). See the [status reference](/api-reference/verify/overview#validation-statuses) for all possible values.
</ResponseField>

<ResponseField name="expiresAt" type="string (ISO 8601)">
  The UTC timestamp at which this validation session expires, approximately 10 minutes from creation. Attempting to verify an expired session returns a `400` error.
</ResponseField>

<ResponseField name="timestamp" type="string (ISO 8601)">
  The UTC timestamp at which this validation was created. This value is used as an input to the HMAC proof calculation.
</ResponseField>

<ResponseField name="proof" type="string">
  An HMAC proof computed from the validation data. Present **only** when your account has a `verificationSecret` configured. You can use this proof with `POST /v1/verify-proof` for offline verification.
</ResponseField>

## Examples

<CodeGroup>

```bash cURL — basic request
curl -X POST https://verify-api.smsmanager.com/v1/validations \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <your-api-key>" \
  -d '{ "phoneNumber": "14155550100" }'
```

```bash cURL — with custom payload
curl -X POST https://verify-api.smsmanager.com/v1/validations \
  -H "Content-Type: application/json" \
  -H "X-API-Key: <your-api-key>" \
  -d '{
    "phoneNumber": "14155550100",
    "payload": { "userId": "user123", "source": "checkout" }
  }'
```

```json Basic request body
{
  "phoneNumber": "14155550100"
}
```

```json Request body with payload
{
  "phoneNumber": "14155550100",
  "payload": {
    "userId": "user123",
    "source": "checkout"
  }
}
```

</CodeGroup>

**Response — without proof** (no `verificationSecret` configured):

```json
{
  "token": "61591bd88d8fbe7736bca535809d1216dd71cb0b5cbf2b07e5a08adfbeb5d35b",
  "status": "pending",
  "expiresAt": "2024-01-01T00:10:00.000Z",
  "timestamp": "2024-01-01T00:00:00.000Z"
}
```

**Response — with proof** (`verificationSecret` configured):

```json
{
  "token": "61591bd88d8fbe7736bca535809d1216dd71cb0b5cbf2b07e5a08adfbeb5d35b",
  "status": "pending",
  "expiresAt": "2024-01-01T00:10:00.000Z",
  "timestamp": "2024-01-01T00:00:00.000Z",
  "proof": "a2c4e6f8b0d2e4f6a8c0e2f4b6d8f0a2c4e6f8b0d2e4f6a8c0e2f4b6d8f0a2"
}
```

## Error responses

| HTTP status | Error code | Description |
|---|---|---|
| `400` | `INVALID_PHONE` | `phoneNumber` is missing, contains non-digit characters, or fails the `^[1-9]\d{6,14}$` pattern check. |
| `400` | `MISSING_FIELD` | A required field (e.g., `phoneNumber`) was not included in the request body. |
| `401` | `MISSING_API_KEY` | The `X-API-Key` header was not present in the request. |
| `401` | `INVALID_API_KEY` | The supplied API key is invalid or has been disabled. |
| `429` | `TENANT_RATE_LIMIT` | Your account has exceeded its validation rate limit. |
| `429` | `PHONE_RATE_LIMIT` | This specific phone number has been submitted too many times in the current window. |
